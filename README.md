# ai-backend-conventions

> AI 에이전트가 일관성 있는 Spring Boot 백엔드 코드를 작성하도록  
> 프로젝트 규칙을 직접 설계하고, 실제 기능 개발에 적용하며 검증하는 토이 프로젝트입니다.

<br />

## 프로젝트 목표

이 프로젝트의 목표는 단순히 Todo 기능을 구현하는 것이 아닙니다.

AI 에이전트를 활용해 백엔드 기능을 개발할 때, 매 요청마다 스타일과 구조가 달라지는 문제를 줄이기 위해 **명확한 프로젝트 규칙을 먼저 정의하고**,  
그 규칙을 실제 기능 구현에 적용하면서 **일관성 있는 코드, 테스트, 문서화 결과를 만들어내는 것**을 목표로 합니다.  

현재는 Todo 도메인을 대상으로 아래 항목을 규칙 기반으로 정리하고 검증했습니다.

- 도메인 중심 패키지 구조
- API 응답 구조 통일
- 예외 처리 규칙
- Controller / DTO / Service / JPA 컨벤션
- 테스트 전략 및 테스트 네이밍 규칙
- Spring REST Docs 기반 API 문서화

인증 기능은 구현이 시작된 상태이지만,  
아직 규칙 정리와 리팩토링이 마무리되지 않아 현재 README 범위에서는 제외합니다.

<br />

## 왜 이 프로젝트를 만들었는가

Cursor, Windsurf 같은 AI 코딩 도구를 사용하면서 느낀 가장 큰 문제는  
**AI가 코드를 잘 만들어주더라도 일관성이 쉽게 무너진다**는 점이었습니다.

같은 프로젝트 안에서도 요청 시점이나 표현 방식에 따라 다음과 같은 차이가 생겼습니다.

- 패키지 구조가 흔들림
- DTO / Entity / Service 책임 분리가 일관되지 않음
- 예외 처리 방식이 매번 달라짐
- 테스트 스타일과 문서화 방식이 통일되지 않음

그래서 이 프로젝트에서는 기능 구현보다 먼저 **프로젝트 규칙을 설계하고**, AI 에이전트가 그 규칙 안에서 코드를 생성하도록 유도하는 방식을 실험했습니다.

즉, 이 프로젝트는 **"AI로 백엔드 기능을 얼마나 빨리 만들 수 있는가"** 보다는 **"AI가 일관성 있는 백엔드 코드를 만들게 하려면 어떤 규칙이 필요한가"** 에 더 가깝습니다.

<br />

## 현재까지 적용한 범위

현재 기준으로 Todo 도메인에 대해 다음 항목을 정리했습니다.

### 1. Todo 도메인 기능 구현
- Todo 생성
- Todo 단건 조회
- Todo 목록 조회
- Todo 수정
- Todo 완료 처리
- Todo 삭제

### 2. 공통 API 응답 구조 적용
- 모든 응답을 `ApiResponse<T>` 구조로 통일
- 성공 / 실패 응답 형식 일관화
- 에러 응답은 `ErrorCode` 기반으로 관리

### 3. 예외 처리 규칙 적용
- 애플리케이션 내부 계층에서는 `CustomException(ErrorCode)` 사용
- 예외 응답은 전역 핸들러에서 일관된 구조로 반환
- 프레임워크 SPI 계층은 프레임워크 계약을 우선하고, 경계에서 프로젝트 예외로 변환

### 4. 레이어별 컨벤션 적용
- Controller는 `Request DTO → ServiceRequest` 변환 후 Service 호출
- Entity와 DTO를 분리
- Entity → Response DTO 변환은 DTO 내부 `from()` 사용
- 트랜잭션은 Service 레이어에서 관리
- 도메인 중심 패키지 구조 유지

### 5. 테스트 전략 적용
- Controller 슬라이스 테스트
- Service 통합 테스트
- Repository 슬라이스 테스트
- Entity 단위 테스트
- E2E 성격의 통합 테스트 분리
- 테스트 메서드 네이밍 규칙 적용

### 6. REST Docs 문서화 적용
- Todo API 문서화 테스트 작성
- AsciiDoc 문서 분리 구조 적용
- 공통 응답 예시와 기능별 문서 분리
- `LocalDateTime` 응답을 API / 문서 모두 ISO-8601 문자열 형식으로 통일

프로젝트 규칙은 도구 전용 단일 파일이 아닌, 역할별 문서로 분리하여 관리합니다.

```text

docs/
  architecture/
    tech-stack.md
    layers.md
  conventions/
    coding.md
    naming.md
    exceptions.md
    testing.md
  api/
    api-response.md
```

각 문서는 AI 에이전트가 참조할 수 있도록 작성되었으며, 기능 개발 시 해당 문서를 컨텍스트로 제공하는 방식으로 활용합니다.
